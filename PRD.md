# NorthLens — Product Requirements Document (PRD)

## Version 1.0 | March 2026

---

## 1. Product Vision

**NorthLens is the competitive intelligence operating system for Canadian small businesses.**

It combines automated web data extraction (inspired by LightFeed) with unified analytics and AI-powered insights (inspired by Triple Whale) into a single platform purpose-built for the Canadian market.

### Core Value Proposition
> "Turn the open web into your competitive advantage — automatically extract competitor data, unify it with your own business metrics, and get AI-powered recommendations that tell you exactly what to do."

### Target Users
- **Primary:** Canadian SMB owners/operators (e-commerce, retail, services) with 1-100 employees
- **Secondary:** Marketing managers, pricing analysts, and business development leads at Canadian companies
- **Tertiary:** Consultants and agencies managing competitive intelligence for multiple Canadian clients

## 2. Product Definition & Glossary

| Term | Definition |
|------|-----------|
| **Pipeline** | An automated data extraction workflow. A user describes what data they want in natural language, NorthLens generates a schema, connects to source URLs, and extracts structured data on a schedule. |
| **Record** | A single extracted data point (e.g., one product listing from a competitor's website). Stored as JSONB with schema-defined fields. |
| **Record Version** | A snapshot of a record at a point in time. When a record changes (e.g., price drops), a new version is created with field-level diffs. |
| **Schema** | The structure definition for a pipeline's data. Generated by AI from natural language, editable by the user. Defines field names, types, and descriptions. |
| **Data Import** | User's own business data (products, prices, costs, inventory) uploaded via CSV or manual entry. Used for comparison against competitor data. |
| **Widget** | A dashboard visualization component (KPI card, chart, feed, opportunity card). Configurable and arrangeable in a grid layout. |
| **Alert** | A conditional trigger that fires when extracted data meets specified criteria (e.g., "notify me when a competitor's price drops below $100"). |
| **AI Advisor** | NorthLens's AI assistant powered by Gemini 2.5 Pro with function calling. It can query data, generate charts, compare records, create briefs, and recommend actions. |
| **Competitive Brief** | An AI-generated weekly intelligence report summarizing market changes, price movements, opportunities, and recommended actions. |
| **NLQ (Natural Language Query)** | The ability to ask questions in plain English and have AI convert them to SQL, execute the query, and return formatted results. |
| **Canadian Context Engine** | A system that enriches all AI interactions with Canadian-specific context: CAD/USD exchange rate, weather, upcoming holidays, seasonal patterns. |
| **Extraction** | The process of fetching a webpage, cleaning the HTML, and using Gemini to extract structured data matching the pipeline's schema. |

## 3. Use Cases

### 3.1 Price Monitoring & Competitive Pricing
**Persona:** Sarah, owner of an outdoor gear e-commerce store in Waterloo, ON

**Scenario:** Sarah wants to track winter jacket prices across Canadian Tire, MEC, and Altitude Sports. She creates a pipeline: "Track men's winter jacket prices, ratings, and availability from these 3 retailers." NorthLens generates a schema (name, price, rating, availability, url), extracts 28 products, and tracks them daily. When Canadian Tire drops their Arctic Parka by 15%, Sarah gets an alert and asks the AI Advisor: "Should I match this price?" The AI queries her import data (margins, inventory), compares market positioning, and recommends: "Hold your price — you're positioned as premium. Instead, create a comparison chart highlighting your waterproof rating advantage."

### 3.2 Market Gap Analysis
**Persona:** Marcus, founder of a specialty coffee subscription service in Toronto

**Scenario:** Marcus tracks competitor offerings across 8 Canadian coffee subscription services. He imports his own product catalog and asks the AI: "Where are the gaps in the market?" The AI analyzes all extracted data and identifies: "No competitor offers a single-origin Ethiopian subscription under $25/mo with free shipping. Your cost structure supports this at 42% margin."

### 3.3 Regulatory & News Monitoring
**Persona:** Priya, compliance officer at a fintech startup in Vancouver

**Scenario:** Priya creates a pipeline to track Canadian regulatory announcements from OSFI, CSA, and Canada.ca. The AI generates a schema for regulation title, agency, date, category, and summary. When new regulations are published, she gets alerts and the AI Advisor generates a brief explaining business impact and required actions.

### 3.4 Review & Sentiment Tracking
**Persona:** Alex, restaurant chain operator with 5 locations in Calgary

**Scenario:** Alex tracks Google reviews and Yelp ratings for his restaurants AND his top 10 competitors. The dashboard shows review trends, average ratings over time, and sentiment analysis. When a competitor's rating drops below 3.5 stars, Alex gets an opportunity alert.

### 3.5 Job Market Intelligence
**Persona:** HR Director at a growing tech company in Ottawa

**Scenario:** Tracks competitor job postings to understand hiring trends, salary ranges, and expansion plans. Pipeline extracts job title, salary range, location, department, and posting date from competitor career pages.

## 4. Feature Specifications

### 4.1 Pipeline Engine

**Purpose:** Transform any website into structured, trackable data using natural language.

**User Flow:**
1. User describes data needs in plain English
2. AI generates an optimized schema (field names, types, descriptions)
3. User reviews/edits schema
4. User adds source URLs
5. User sets schedule (hourly/daily/weekly/manual)
6. Pipeline runs first extraction
7. Data appears in tables, dashboard updates

**Capabilities:**
- AI-powered schema generation from natural language (Gemini 2.5 Flash)
- Multi-URL extraction with deduplication
- Automatic change detection with field-level diffs
- Version history for every record
- Content hash-based deduplication
- HTML cleaning and LLM-based extraction
- Support for list mode (multiple items) and detail mode (single item)
- Scheduled re-extraction with configurable frequency

### 4.2 Data Tables

**Purpose:** View, search, filter, sort, and export extracted data.

**Capabilities:**
- Dynamic column generation from pipeline schema
- Server-side pagination (20/50/100 rows per page)
- Multi-column sorting
- Full-text client-side search across all fields
- CSV export with proper formatting
- Type-aware cell rendering (currency, URL, date, rating)
- Change indicators for versioned records

### 4.3 Dashboard

**Purpose:** At-a-glance competitive intelligence with customizable widgets.

**Widgets:**
- **KPI Cards** — Numeric value + sparkline + trend indicator (your avg price, market avg, record count)
- **Price Tracker** — Multi-line chart comparing your prices to market averages over time
- **Recent Changes Feed** — Timeline of detected data changes with field-level detail
- **Opportunity Card** — AI-generated insight about market gaps or strategic actions
- **Chart Widget** — Configurable line/bar charts from pipeline data

### 4.4 AI Advisor

**Purpose:** Context-aware AI assistant that can query data, generate charts, and recommend actions.

**Capabilities:**
- Conversational interface with persistent chat history
- Full business context injection (profile, pipelines, Canadian context)
- Function calling: execute_sql, search_records, create_chart, get_price_history, compare_records
- Markdown-formatted responses with inline data
- Suggested prompts for common queries
- Chat session management (new chat, history)

**Powered by:** Gemini 2.5 Pro with function calling + Backboard.io for persistent memory

### 4.5 Natural Language Query (NLQ)

**Purpose:** Ask questions in English, get SQL + results instantly.

**User Flow:**
1. User types question: "Show me all products under $100 from Canadian Tire"
2. AI generates SQL query with explanation
3. SQL appears in Monaco editor (editable)
4. Results execute and display in table format
5. User can modify and re-run

**Security:** Only SELECT statements allowed. Mutation keywords blocked at both AI and execution layers.

### 4.6 Data Import

**Purpose:** Bring your own business data into NorthLens for comparison.

**Capabilities:**
- CSV upload with drag-and-drop
- Auto-type detection (currency, number, URL, text)
- Column preview with first 5 rows
- Configurable column type mapping
- Batch record insertion

### 4.7 Alert System

**Purpose:** Get notified when competitive data meets specified conditions.

**Capabilities:**
- Field-level condition triggers (greater than, less than, equals, contains)
- Pipeline-scoped alerts
- Enable/disable toggle
- Alert event history with timestamps
- In-app notification center

### 4.8 SQL Console

**Purpose:** Power-user interface for direct data querying.

**Capabilities:**
- Monaco code editor with SQL syntax highlighting
- NLQ bar for natural language → SQL conversion
- Query execution with result table display
- Execution time tracking
- SELECT-only security enforcement

### 4.9 Canadian Context Engine

**Purpose:** Enrich all AI interactions with Canadian-specific business intelligence.

**Data Sources:**
- Live CAD/USD exchange rate (Open Exchange Rate API)
- Current weather in user's location (Open-Meteo, no API key)
- Upcoming Canadian holidays (2026 calendar)
- Current season detection
- Provincial context

### 4.10 Command Palette (⌘K)

**Purpose:** Power-user navigation and action execution.

**Capabilities:**
- Keyboard shortcut (⌘K / Ctrl+K)
- Search across all pages
- Quick navigation to any pipeline's data
- Action shortcuts (new pipeline, open AI advisor)

## 5. Non-Functional Requirements

### Performance
- Dashboard load: <2s on 3G connection
- Table rendering: 100+ rows at 60fps (TanStack Table virtualization)
- AI chat response: <5s for non-tool-calling queries
- Pipeline extraction: <30s per URL

### Security
- Row-Level Security (RLS) on all 12 database tables
- Users can only access their own data
- SQL console restricted to SELECT-only queries
- API keys stored as environment variables, never in client code
- Supabase Auth with email/password + OAuth

### Scalability
- PostgreSQL with JSONB for flexible schemas
- pgvector for future semantic search
- Edge Functions for serverless compute
- Supabase Realtime for live updates

### Accessibility
- Dark mode optimized for extended use
- Keyboard navigation throughout
- ARIA labels on interactive elements
- Responsive design (mobile → desktop)
